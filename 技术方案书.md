## 一、 项目概述

### 1.1 项目背景
大学新生报到数字化管理系统，从MVP（Flask+SQLite）升级为企业级架构（Django+PostgreSQL+DRF）。

### 1.2 核心目标
- 构建可扩展的RESTful API
- 支持高并发访问
- 提供完整的Admin管理后台
- 实现JWT无状态认证
- 自动生成API文档

### 1.3 技术栈升级对比

| 组件 | MVP版本 | 企业版本 |
|------|---------|----------|
| Web框架 | Flask 3.0 | Django 5.0.4 |
| 数据库 | SQLite | PostgreSQL 14+ |
| ORM | SQLAlchemy | Django ORM |
| API | 无（服务端渲染） | Django REST Framework |
| 认证 | Session | JWT (Simple JWT) |
| 文档 | 无 | Swagger (drf-yasg) |
| 后台 | 无 | Django Admin |
| 跨域 | 不适用 | CORS Headers |

## 三、 数据层设计

### 3.1 数据库配置

**数据库类型**: PostgreSQL 14+  
**连接方式**: psycopg2-binary  
**配置示例**:
`python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'freshman_db',
        'USER': 'postgres',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '5432',
    }
}
`

### 3.2 数据模型

#### 3.2.1 User模型（扩展Django User）

Django内置User表字段：
- id: 主键
- username: 用户名（学号）
- password: 密码（加密）
- email: 邮箱
- is_active: 是否激活
- date_joined: 注册时间

#### 3.2.2 StudentProfile模型（学生档案表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK | 主键 |
| user | OneToOne | FK, unique | 关联User |
| student_id | Char(20) | Unique | 学号 |
| real_name | Char(50) | NOT NULL | 真实姓名 |
| phone | Char(20) | NOT NULL | 联系电话 |
| dorm_info | Char(100) | NULL | 宿舍信息 |
| status | Char(20) | DEFAULT | pending/done |
| report_time | DateTime | NULL | 报到时间 |
| created_at | DateTime | Auto | 创建时间 |
| updated_at | DateTime | Auto | 更新时间 |

**表关系**:
`
User (1)  (1) StudentProfile
   一对一关系，通过user_id外键关联
`

## 四、 业务层设计

### 4.1 RESTful API设计

#### 4.1.1 API路由清单

| 路由 | 方法 | 功能 | 认证 |
|------|------|------|------|
| /api/auth/login/ | POST | JWT登录 | 否 |
| /api/auth/refresh/ | POST | 刷新Token | 否 |
| /api/students/profile/ | GET | 获取档案 | 是 |
| /api/students/profile/ | PUT | 更新档案 | 是 |
| /api/students/report/ | PATCH | 提交报到 | 是 |
| /admin/ | GET | 管理后台 | 是 |
| /swagger/ | GET | API文档 | 否 |

#### 4.1.2 API响应格式

**成功响应**:
`json
{
  "id": 1,
  "student_id": "2024001",
  "real_name": "张三",
  "phone": "13800138000",
  "dorm_info": "1号楼302",
  "status": "done",
  "report_time": "2026-04-25T10:30:00Z"
}
`

**错误响应**:
`json
{
  "error": "ValidationError",
  "message": "手机号格式错误",
  "details": {
    "phone": ["请输入有效的手机号"]
  }
}
`

### 4.2 JWT认证机制

**认证流程**:
`
1. 客户端POST /api/auth/login/
   Body: {username, password}
   
2. 服务器验证凭证
   
3. 返回JWT Token
   {
     "access": "eyJ0eXAi...",
     "refresh": "eyJ0eXAi...",
     "expires_in": 7200
   }
   
4. 客户端后续请求携带Token
   Header: Authorization: Bearer <access_token>
   
5. Token过期后使用refresh获取新Token
`

**JWT配置**:
`python
SIMPLE_JWT = {
    'ACCESS_TOKEN_LIFETIME': timedelta(hours=2),
    'REFRESH_TOKEN_LIFETIME': timedelta(days=7),
    'ROTATE_REFRESH_TOKENS': True,
    'ALGORITHM': 'HS256',
}
`

### 4.3 核心业务流程

#### 4.3.1 用户注册流程
`
Admin后台创建用户
  
自动触发Signal
  
创建StudentProfile（status=pending）
  
返回用户信息
`

#### 4.3.2 报到流程
`
1. 用户登录获取JWT Token
  
2. GET /api/students/profile/
   查看当前状态
  
3. PATCH /api/students/report/
   提交: {real_name, phone, dorm_info}
  
4. 服务器更新:
   - status = 'done'
   - report_time = now()
  
5. 返回更新后的档案
`

## 七、 安全设计

### 7.1 密码安全
- Django内置PBKDF2加密
- 密码哈希不可逆
- 强制密码复杂度验证

### 7.2 JWT安全
- Token有效期2小时
- Refresh Token 7天
- 支持Token黑名单
- HTTPS传输（生产环境）

### 7.3 CORS跨域
`python
CORS_ALLOWED_ORIGINS = [
    'http://localhost:3000',
    'http://127.0.0.1:3000',
]
`

### 7.4 SQL注入防护
- Django ORM参数化查询
- 禁止原生SQL拼接

### 7.5 XSS防护
- Django模板自动转义
- DRF数据验证
